## 原始版本 → 模块化重构

原始系统是一个 Jupyter notebook (`sz_codes.ipynb`)，包含数据下载、指标计算、信号筛选、回测的全部逻辑。重构后拆分为以下独立模块：

| 模块 | 职责 |
|------|------|
| `config.py` | 全局配置常量 |
| `universe.py` | 股票池定义（深交所代码段、板块判断） |
| `downloader.py` | yfinance 数据下载与本地缓存 |
| `indicators.py` | 技术指标计算（RSI、CRSI、ADX、CCI、OBV 等） |
| `labels.py` | 涨停标签构建（用于 ML） |
| `strategies.py` | 信号函数定义（统一接口 `signal_xxx(df) -> Series[bool]`） |
| `screener.py` | 组合筛选策略（多信号 AND 组合） |
| `backtest.py` | 回测引擎 |
| `agent.py` | 模拟交易 Agent（三段式卖出、涨停续板学习） |
| `diagnose.py` | 特征诊断（涨停 vs 非涨停对比分析） |
| `optimizer.py` | 信号组合优化器 |
| `visualize.py` | K线图、资金曲线、回撤图 |
| `pipeline.py` | 数据流水线（初始化 / 每日更新） |
| `run.py` | CLI 入口 |
| `web/` | 静态网站生成器 |

## 关键优化记录

### 1. OBV涨停梦信号优化

**问题**：原始 OBV涨停梦只有 OBV 突破 + 量能堆积 2 个信号，筛选出的股票太多，涨停命中率极低（549 笔交易仅 16 笔涨停，2.9%）。

**改动**：通过 `diagnose.py` 对涨停/非涨停交易的特征做统计对比（Cohen's d 排序），新增 2 个过滤条件：

| 过滤 | 依据 | 效果 |
|------|------|------|
| CCI > 50 | 动量确认，与OBV/量能正交（d=1.41） | 保留87.5%涨停 |
| 日涨幅 > 2% | 价格验证OBV突破（涨停组均值4.5% vs 非涨停1.3%） | 进一步筛除弱信号 |

> 注：早期版本曾排除创业板（20%涨停板更难触发），后经 240 天数据验证创业板涨停率（1.0%）与主板（1.3%）接近，且该策略收益主要来自 2-8% 波段而非涨停，因此移除了创业板过滤，保留全市场选股。

**涉及文件**：
- `config.py`：新增 `CCI_MOMENTUM_FLOOR = 50`、`DAILY_GAIN_GT2_PCT = 0.02`
- `strategies.py`：新增 `signal_cci_momentum_floor`、`signal_daily_gain_gt2`
- `screener.py`：`screen_obv_momentum` 新增两个信号

### 2. 交易真实性约束

**问题**：回测结果虚高——没有考虑两个 A 股基本规则。

#### 2a. 涨停股无法买入

当日收盘价达到涨停价的股票处于封板状态，实际无法买入。之前的回测"买入"了这些股票，收益是虚假的。

**改动** (`agent.py` → `_decide_buy`)：
```python
# 判断当日是否涨停
limit_price = round(prev_close * (1 + ratio), 2)
if close >= limit_price * (1 - LIMIT_TOLERANCE):
    return False, '当日涨停无法买入'
```

**影响**：某些激进策略（如日涨>5%筛选）的"利润"从 +14% 变为 -8%，证明其收益完全来自买入涨停股的虚假交易。

#### 2b. 印花税

A 股卖出时收取千分之一印花税，之前未计入。

**改动**：
- `config.py`：新增 `STAMP_TAX_RATE = 0.001`
- `agent.py` → `_execute_sell`：卖出时扣除印花税

### 3. ADX 过滤器（基于亏损分析）

**问题**：加入真实性约束后，OBV涨停梦 42 笔交易中仍有较多大亏损（跌幅>3%）。

**方法**：不再试图预测赢家，而是分析输家（跌幅>3%的交易）寻找可过滤的共同特征。

**发现**：ADX 是最强判别因子——
- ADX ≥ 40 的交易：67% 大亏损率（趋势过强 = 末期反转风险高）
- ADX < 40 的交易：32% 大亏损率

**改动**：
- `config.py`：新增 `ADX_BUY_MAX = 40`
- `agent.py` → `_decide_buy`：ADX ≥ 40 时拒绝买入

**效果**（60天回测）：

| 指标 | 无ADX过滤 | 有ADX过滤 |
|------|-----------|-----------|
| 交易数 | 42 | 21 |
| 胜率 | 47.6% | 66.7% |
| 总收益 | +0.49% | +1.14% |
| 最大回撤 | -1.77% | -1.14% |

多时间窗口验证（60/120/240天）均保持正收益和可控回撤。

### 4. 恢复创业板选股

**背景**：首轮优化曾排除创业板（300/301），理由是 20% 涨停板更难触发。

**验证**：240 天数据对比创业板与主板的次日涨停率（1.0% vs 1.3%），差异不显著。同时加入创业板后交易量翻倍，收益显著提升：

| 指标 | 仅主板(60天) | 含创业板(60天) |
|------|-------------|---------------|
| 交易数 | 21 | 51 |
| 胜率 | 66.7% | 60.8% |
| 总收益 | +1.14% | +3.83% |
| 夏普比率 | 1.14 | 3.07 |

**改动**：`screener.py` 的 `screen_obv_momentum` 移除创业板前置过滤，改为全市场选股。

### 6. `--screen` 参数支持

**改动**：
- `run.py`：`agent` 命令新增 `--screen` 参数，可选择不同筛选策略
- `agent.py`：构造函数接受 `screen_func` 参数

```bash

```

### 5. ADX 过滤独立为 OBV波段策略

**背景**：ADX<40 过滤器在波段交易中效果好（提高胜率、降低回撤），但会过滤掉高 ADX 的涨停候选股。60 天内有 2 笔次日涨停的候选被 ADX 过滤器拦截（002156 ADX=48、301079 ADX=53）。

**调整**：
- OBV涨停梦保持原始定位：T0 买入博 T+1 涨停，**不加 ADX 过滤**
- 新增 OBV波段策略：OBV涨停梦信号 + ADX<40，用于 2-8% 波段交易

**改动**：
- `agent.py`：`_decide_buy` 移除 ADX 过滤（改由 screener 层控制）
- `strategies.py`：新增 `signal_adx_below_max`
- `screener.py`：新增 `screen_obv_wave`（OBV波段），注册到 `ALL_SCREENS`

```bash
python run.py agent --days 60 --screen OBV涨停梦   # 涨停策略（无ADX过滤）
python run.py agent --days 60 --screen OBV波段      # 波段策略（ADX<40）
```

## 核心教训

1. **不能买涨停股**：这是 A 股最基本的约束。任何忽略此规则的回测都会高估收益。
2. **预测赢家不如过滤输家**：涨停预测样本太少容易过拟合；反向分析大亏损交易的特征更可靠。
3. **简单规则胜过复杂模型**：单一 ADX<40 过滤器的效果优于 6 维指标分桶评分系统。
4. **多时间窗口验证**：单一回测窗口容易误导，应在不同周期确认策略稳定性。
